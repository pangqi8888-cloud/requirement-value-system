## 📊 评估维度详解

### 1. 商业价值（权重 30%）
**评分依据：**
- 预期收益描述的详细程度
- 关键词识别：收入、营收、利润、市场份额、转化率、留存、GMV
- 业务背景的完整性

**示例：**
- 高分：详细描述预期增加 500 万营收，提升 20% 转化率
- 低分：简单描述"提升用户体验"

### 2. 用户影响（权重 25%）
**评分依据：**
- 影响用户数量
  - >100,000 用户：+40 分
  - >10,000 用户：+30 分
  - >1,000 用户：+20 分
- 目标用户描述的清晰度

**示例：**
- 高分：影响 50,000 企业用户，明确目标群体
- 低分：影响 1,000 用户，目标模糊

### 3. 实现成本（权重 20%）
**评分依据：**
- 成本越低，分数越高
- 关键词识别：
  - "高"、"复杂"、"困难"：-30 分
  - "中"、"适中"：-15 分
  - "低"、"简单"、"容易"：+10 分

**示例：**
- 高分：实现成本低（1-3 人天）
- 低分：实现成本高（10+ 人天）

### 4. 紧急程度（权重 15%）
**评分依据：**
- 紧急等级 1-5 线性映射到 20-100 分
- 等级 5（非常紧急）：100 分
- 等级 1（不紧急）：20 分

### 5. 竞品对比（权重 10%）
**评分依据：**
- 关键词识别：
  - "领先"、"超越"：+30 分
  - "对齐"、"追平"：+20 分
  - "落后"、"缺失"：+40 分（竞品有我们没有，优先级更高）

**示例：**
- 高分：竞品已有此功能，我们落后
- 低分：无竞品信息

## 💡 使用建议

### 如何获得高分？

1. **详细填写预期收益**
   - 量化指标：提升 XX% 转化率
   - 具体金额：增加 XX 万营收
   - 使用关键词：收入、利润、市场份额

2. **明确影响用户数**
   - 尽可能准确估算
   - 数量越大得分越高

3. **选择合适的实现成本**
   - 如实评估
   - 成本低的需求更容易获得高分

4. **提供竞品信息**
   - 说明竞品功能情况
   - 特别是"竞品有我们没有"的情况

5. **设置合理的紧急程度**
   - 根据实际情况设置
   - 不要所有需求都设为最高

## 🔧 技术细节

### 评分算法核心代码
位置：`backend/app/services/ai_evaluation.py`

```python
# 综合得分计算
total_score = (
    business_value_score * 0.30 +  # 商业价值
    user_impact_score * 0.25 +     # 用户影响
    cost_score * 0.20 +            # 实现成本
    urgency_score * 0.15 +         # 紧急程度
    competitor_score * 0.10        # 竞品对比
)
```

### API 接口
位置：`backend/app/api/requirements.py`

主要接口：
- `POST /api/v1/requirements/` - 创建需求并自动评估
- `GET /api/v1/requirements/` - 获取需求列表（支持排序）
- `PUT /api/v1/requirements/{id}` - 更新需求并重新评估

### 前端组件
- `RequirementForm.jsx` - 需求创建表单
- `ScoreCard.jsx` - 评分可视化组件
- `RequirementList.jsx` - 需求列表主页面
